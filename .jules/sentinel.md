## 2024-03-05 - Directory Traversal Bypass via URL-Encoded Sequences
**Vulnerability:** The `resolve_local_path` function in `scripts/validate_pages.py` was vulnerable to directory traversal. While it resolved paths to a candidate, an attacker could supply URL-encoded sequences (e.g., `%2e%2e%2f` for `../`) which when decoded by `unquote` could escape the intended repository root (`ROOT`).
**Learning:** URL-encoded strings can bypass standard path checks if those checks are only performed on the raw (encoded) string. Validating only the encoded version isn't enough; the fully decoded and normalized path must also be verified against the intended boundary.
**Prevention:** Always apply path resolution to an absolute path (`os.path.abspath`) before performing boundary checks (e.g., `os.path.commonpath([ROOT, path]) == ROOT`). This ensures no `ValueError` is raised from mixing absolute and relative paths. Perform these checks on both the raw path and the explicitly URL-decoded path to prevent bypasses, and ensure any failure falls back to a non-existent path (like `/dev/null`) instead of returning a valid directory which would be considered "found" during existence checks.
