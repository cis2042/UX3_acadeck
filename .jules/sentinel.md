## 2026-03-06 - [Reject Fix for validate_pages.py (Security Theater)]
**Vulnerability:** A script `scripts/validate_pages.py` was checking dead links using local paths. The previous assumption was that an attacker could provide a malicious `href` value in an `index.html` causing a directory traversal check.
**Learning:** This was identified as "security theater" because this script is run locally/in CI on repository files, not on untrusted user input from an external attacker. Furthermore, fixing local path joining with `os.path.commonpath` can cause scripts to fail with `ValueError` when run in environments where `ROOT` might evaluate to a relative path. Security tools and fixes must always apply to actual attack surfaces where untrusted data is processed. Adding complex safety checks to build/validation scripts that run on trusted repository code offers no real security benefit and can break CI pipelines.
**Prevention:** Do not add "security fixes" to internal build/validation scripts unless those scripts specifically ingest untrusted external data. Verify the attack surface before implementing complex validations like `os.path.commonpath`.
